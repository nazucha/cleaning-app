# スクリプト改善内容

## 問題点

MSZプレフィックスには約510個の型番があり、処理に非常に時間がかかっていました。また、`fetchHTML`関数にタイムアウトがないため、ネットワークエラーやサーバーの応答が遅い場合、スクリプトが無限に待機する可能性がありました。

## 改善内容

1. **タイムアウト処理の追加**
   - `fetchHTML`関数に10秒のタイムアウトを追加
   - タイムアウト時はエラーを投げて次の型番に進む

2. **進捗表示の改善**
   - 各プレフィックスの処理開始時に型番数を表示
   - 10件ごとに進捗を表示（大量の型番がある場合）

## 実行方法

改善されたスクリプトを再実行してください：

```bash
cd /Users/malove/Desktop/再チャレンジ/cleaning-app

# 三菱電機を再実行
node scripts/scrapeMitsubishiFullModels.js
```

または、一括実行スクリプトを再実行：

```bash
./実行スクリプト_自動.sh
```

## 注意事項

- MSZのような大量の型番があるプレフィックスは、処理に時間がかかります（約510件 × 4カテゴリー × 0.5秒 = 約17分以上）
- タイムアウトが発生した場合は、その型番はスキップされて次の型番に進みます
- 進捗表示により、処理が進行していることが確認できます
