# サーバー起動確認

## 問題

AIによる概要の判定が機能していない原因：
1. **サーバーが起動していない**（`ERR_CONNECTION_REFUSED`）
2. フロントエンドがポート3000に接続しようとしているが、サーバーはポート3001で起動している

## 対応

### 1. フロントエンドの修正 ✅ 完了

- 関数名を`analyzeModelWithChatGPT`から`analyzeModelWithServer`に変更
- ポートを3000から3001に変更
- ログメッセージを「ChatGPT」から「AI」に変更

### 2. サーバーの起動

ターミナルで以下を実行：

```bash
cd /Users/malove/Desktop/再チャレンジ/cleaning-app
node server.js
```

**確認事項**:
- サーバーがポート3001で起動しているか確認
- ログに `[サーバー] Gemini API呼び出し: ... (Vertex AI)` と表示されれば成功

### 3. 動作確認

1. サーバーを起動
2. ブラウザでアプリを開く
3. 型番を入力（例: `AJT-SEP`）
4. AIによる概要が表示されるか確認

## トラブルシューティング

### ポート3001が使用中の場合

`.env`ファイルで別のポートを指定：

```env
PORT=3002
```

フロントエンドの`App.tsx`も同じポートに変更：

```typescript
const API_ENDPOINT = "http://localhost:3002/api/analyze-model";
```
