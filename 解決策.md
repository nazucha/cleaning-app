# PlatformConstantsエラーの解決策

## 問題の原因
1. SDK 54の新しいアーキテクチャ（Bridgeless mode）とExpo Goアプリの互換性問題
2. 新しいExpo GoはSDK 55のみをサポートする予定

## 解決方法

### 方法1: 開発ビルドを使用（推奨）
Expo Goではなく、開発ビルドを使用することで、SDK 54の新しいアーキテクチャを正しくサポートできます：

```bash
# iOSシミュレーターで起動
npx expo run:ios

# または、実機で起動
npx expo run:ios --device
```

### 方法2: SDK 51に戻す（一時的な解決策）
Expo GoアプリがSDK 54を要求しているため、この方法は使えません。

### 方法3: Web版で動作確認
Web版ではこのエラーは発生しません。まずはWeb版で動作確認してください。

### 方法4: Expo Goアプリの自動更新を無効化
警告メッセージに従って、App Storeの設定で自動更新を無効化し、現在のExpo Goアプリを使用し続ける。

## 推奨される次のステップ
1. まずはWeb版で動作確認
2. 動作が確認できたら、開発ビルドを作成
3. 実機にインストールしてテスト
