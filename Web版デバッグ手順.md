# Web版デバッグ手順

## 現在の状況
- メタディスクリプションタグ: ✅ 追加済み
- アプリのコード: ✅ 正常
- 空白ページの問題: 🔍 調査中

## デバッグ手順

### 1. ブラウザのコンソールを確認
1. ブラウザで `http://localhost:8081` を開く
2. F12 または Cmd + Option + I（Mac）で開発者ツールを開く
3. **Consoleタブ**を確認
   - エラーメッセージ（赤色）がないか確認
   - "App component rendered" というログが表示されているか確認
   - "Platform.OS: web" というログが表示されているか確認

### 2. ElementsタブでHTMLを確認
1. 開発者ツールの**Elementsタブ**（または**Inspector**）を開く
2. `<body>`タグを展開
3. `<div id="root">`の中身を確認
   - 中身が空の場合 → JavaScriptエラーの可能性
   - 中身がある場合 → CSSの問題の可能性

### 3. Networkタブでリソースを確認
1. 開発者ツールの**Networkタブ**を開く
2. ページをリロード（Cmd + R）
3. 以下のリソースが正しく読み込まれているか確認：
   - `AppEntry.bundle?platform=web` → ステータスが200（成功）か確認
   - エラー（赤色）が表示されていないか確認

### 4. 確認すべきエラーメッセージ
以下のエラーが表示されている場合は、対応が必要です：

- **"Cannot find module"** → パッケージが不足している
- **"ReferenceError"** → 変数や関数が定義されていない
- **"TypeError"** → 型エラー
- **"Network request failed"** → ネットワークエラー

## 次のステップ
ブラウザのコンソールに表示されているエラーメッセージを共有してください。
それに基づいて修正を行います。
