# 取扱説明書データ統合手順

## 概要

取扱説明書から取得したお掃除機能データを`cleaningFunctionData.json`に統合し、アプリで優先的に使用します。

## 手順

### 1. スクリプト実行完了を確認

全メーカーのスクリプトが完了するまで待機してください。

```bash
# 進捗確認
tail -f logs/all_manufacturers_*.log
```

### 2. 取扱説明書データファイルの確認

```bash
cd /Users/malove/Desktop/再チャレンジ/cleaning-app

# 取扱説明書から取得したデータファイルを確認
ls -lh data/*CleaningFunctionFromManual.json
```

### 3. データ統合スクリプトの実行

取扱説明書のデータファイルが作成されたら、統合スクリプトを実行します：

```bash
cd /Users/malove/Desktop/再チャレンジ/cleaning-app
node scripts/mergeCleaningFunctionFromManual.js
```

このスクリプトは：
- 取扱説明書から取得したデータを`cleaningFunctionData.json`に統合
- 取扱説明書のデータを優先的に使用
- 既存データで取扱説明書にない型番は保持

### 4. アプリの確認

ブラウザでアプリをリロード（Ctrl+Shift+R または Cmd+Shift+R）して、取扱説明書のデータが優先的に使用されることを確認してください。

## データの優先順位

アプリでは以下の優先順位でお掃除機能の有無を判定します：

1. **取扱説明書から取得したデータ**（最優先）
   - `cleaningFunctionData.json`の`source: 'manual'`のデータ
   
2. **ローカルの判定ロジック**（フォールバック）
   - 型番パターンによる判定
   - キーワードによる判定

## 注意事項

- 取扱説明書からデータが取得できなかった型番は、既存のローカル判定ロジックが使用されます
- 取扱説明書のデータは、スクリプト実行中に自動的に取得されます
- 統合スクリプトは、取扱説明書のデータファイルが作成された後に実行してください
